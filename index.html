How We Use Your Data

Your data is used exclusively to generate and update your BOQ Google Sheet. We do not sell, share, or transmit your data to any third party other than Google (for the purpose of writing to your Google Sheet) and Autodesk (via the Revit API). No data is stored on SUN_SLDev servers.

Data Storage

All project data is stored locally on your machine in a JSON configuration file alongside your Revit project file. Google authentication tokens are stored locally in C:\ProgramData\BOQConnect\tokens\ on your machine. No data is uploaded to or stored on any SUN_SLDev infrastructure.

Google API Access

BOQ Connect requests the following Google permissions: create and edit Google Sheets files it creates, create folders in your Google Drive, and read your Google account email and name. These permissions are used solely within the context of your BOQ project. You can revoke access at any time via your Google Account settings at myaccount.google.com.

Data Retention

You can delete the configuration file and token folder at any time to remove all locally stored data. Deleting the Google Sheet from your Drive removes all cloud-stored BOQ data.

Help & Getting Started

How does BOQ Connect work?

BOQ Connect reads element data directly from your Revit model. You fill in 5 shared parameters on each element type in Revit (Item Number, Description, Unit, Rate, and optionally Measure By). Every time you save the Revit file, quantities are extracted and your Google Sheet updates automatically.

Setting up element types in Revit

After setup, open any element's Type Properties in Revit. You will see the BOQ Connect parameters under the Data group. Fill in the Item Number, Description, Unit, and Rate for each element type you want in the BOQ.

Highlight colours in the Sheet

Amber rows — one or more BOQ parameters are missing. Go to that element type in Revit and fill them in.
Red rows — this element type was in a previous sync but has been deleted from the model. Review and delete the row manually if confirmed.

Missing rate or description

All data must be set in Revit Type Properties — not in the Google Sheet. The Sheet is a read-only output. Update the parameters in Revit and save to refresh.
